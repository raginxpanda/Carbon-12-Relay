RELAY 0.2.11 — send-loop resilience (from Fable's review)
=========================================================
Only src/client.mjs changed. Drag it into your project -> Replace -> Yes.
Pairs with BOT r258. Stacks on all prior drops.

FIXES (the "deferred forever" class of bug that ate a debugging session)
 • BACKOFF: a failing batch now backs off (1s,2s,4s...60s) instead of retrying
   every 5s forever.
 • 413 SPLIT: an oversize batch is split in half and retried smaller, instead of
   looping on the same too-big body (the 16KB-limit bug in general form).
 • DEAD-LETTER: after N consecutive failures the poison batch is set aside (with a
   callback so the UI can show "N events couldn't sync"), so one bad batch can't
   dam the whole pipe.
 • 401 NO LONGER NUKES THE QUEUE: a transient bad token during a big catch-up used
   to silently discard everything queued. Now it keeps the events and backs off.
 • Richer health(): failStreak, deadLettered, backingOff — for surfacing in the UI.

NOTE: the onDeadLetter callback is available if you want to wire a UI toast
("N events couldn't sync — copy log"); main.cjs can pass it when constructing
clients. Not required — the resilience works without it.
