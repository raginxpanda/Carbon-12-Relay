RELAY 0.2.23 — "Restart & Update" now actually restarts
=======================================================
THE BUG: clicking "Restart & Update" closed the app (or tried to) but did NOT
relaunch into the new version.

ROOT CAUSE: the old install path had a hard `app.exit(0)` fallback firing 4s after
quitAndInstall. app.exit is a HARD kill — it was racing the installer and killing
the relaunch handoff before it could bring the app back. It also used the
non-silent installer (isSilent=false), whose UI can interrupt the auto-relaunch.

THE FIX (reliable restart):
 • quitAndInstall(true, true) — SILENT install (no UI to interrupt) + forceRunAfter
   (relaunch after install). This is the combo that restarts cleanly on Windows.
 • autoInstallOnAppQuit = true — if the direct path ever fails, the update still
   installs on the next quit.
 • Fallback now uses app.relaunch() + app.quit() (GRACEFUL) instead of app.exit()
   (HARD kill) — a graceful quit preserves the relaunch; a hard exit killed it.
 • The 8s safety net also uses app.quit(), never app.exit().
 • Teardown (stop relay, destroy tray, strip hide-to-tray close handler) is
   unchanged — that part correctly lets the app close.

CONSOLIDATED electron set (carries everything: allowlist, live-count, update
button + auto-popup, close/teardown, collapsible modules, QoL batch, catalog
search+sort). Supersedes 0.2.13–0.2.22 electron files.

IMPORTANT — the catch-22 again: the version installed on your machine RIGHT NOW
still has the old (broken) restart code. So updating TO 0.2.23 may still not
auto-restart — you may need to reopen the app manually this once. From 0.2.23
FORWARD, "Restart & Update" restarts cleanly on its own. (The fix can only run
once it's the installed version.)
