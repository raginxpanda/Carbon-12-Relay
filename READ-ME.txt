RELAY 0.2.17 — update button now actually closes the app (fix)
==============================================================
THE BUG you hit: clicked "Restart & Update", it said to close the app, you kept
clicking Retry, and it wouldn't close.

ROOT CAUSE: quitAndInstall was called while the app was still holding itself open:
 • the relay's file tail + flush timer were still running,
 • the tray icon was still alive,
 • and the window had a close-interceptor (win.on('close') -> hide to tray) that
   actively PREVENTED it from closing.
So Windows couldn't swap the .exe, and the installer sat on the close/retry prompt.

THE FIX: installUpdate now fully tears down before quitAndInstall —
 1) stops the relay (releases the log file + timers),
 2) destroys the tray,
 3) strips the hide-to-tray close handler and destroys all windows,
 4) then calls quitAndInstall(isSilent=false, forceRunAfter=true),
 5) HARD FALLBACK: force-exits after 4s if anything's still alive, so the
    installer's file lock always clears. (quitAndInstall relaunches the new version.)
Also: window-all-closed now allows the quit while an update is in progress.

CONSOLIDATED: this drop is the current complete electron/ set —
 • main.cjs: pairing allowlist + live-count refresh + update button + THIS fix
 • index.html: recent-ordering fix (0.2.16)
 • preload.cjs: update bridges (0.2.15)
Shipping this supersedes 0.2.13–0.2.16 for the electron/ files.
(src/relay.mjs from 0.2.14 is still needed for the live-count fix if not yet shipped.)
