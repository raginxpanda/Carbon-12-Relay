====================================================

sends ~45 unique blueprints instead of 1710 repeats. (It already worked correctly
because the BOT dedupes on its side — this just makes the relay stop wasting the
bandwidth. Your /blueprint mine count does not change; the "sent" number will.)

  2. Drag "src" into your project folder -> Replace src\backfill.mjs -> Yes.
  3. Ship:  npm version patch  then  git push --follow-tags   (publish the draft release)
     or test now:  npm run dist  -> install.

AFTER: hit Catch up again — it should now say "Synced ~45" instead of 1710.
(Re-running is safe: bot ignores dupes via ON CONFLICT.)

NOTE ON NOTIFICATIONS (separate from this fix)

  (it tails Game.log live). If you earned it while the relay was off, catch-up
  picks it up but no toast fires. Also check Windows notifications aren't muted
  (Focus Assist / Do Not Disturb off, notifications allowed for Carbon-12 Relay).
  Test: with the relay running, earn a BP and watch the Activity log for
  "blueprint earned: <name>" — if that shows but no toast, it's a Windows setting.
