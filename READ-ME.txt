RELAY 0.2.13 — pairing endpoint allowlist (security, from Fable's review)
=========================================================================
Only electron/main.cjs changed. Drag into your project -> Replace -> Yes.

FIX — the only real remote-facing hole in the relay
 The carbon12://pair deep link can be triggered by ANY webpage, with an
 attacker-chosen label and endpoint. Before, it would happily pair you to an
 attacker's server (streaming your telemetry there / crediting your data to
 their org). Now:
  • Endpoint host is ALLOWLISTED to carbon-12.gg (and www.). Anything else is
    BLOCKED with a warning, unless you explicitly enable custom endpoints.
  • The confirm dialog visually distinguishes "official server" (normal) from
    "CUSTOM SERVER" (loud warning, defaults to Cancel).
  • Blocks lookalike domains (carbon-12.gg.evil.com etc.) — verified.

 To allow a custom endpoint (self-hosters): set allowCustomEndpoint:true in your
 relay config. Default is off, which is right for everyone using the official server.
